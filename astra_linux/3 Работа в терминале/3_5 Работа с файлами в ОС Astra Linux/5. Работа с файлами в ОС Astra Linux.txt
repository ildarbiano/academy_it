========== Файловая система в Linux ==============================================================================================
организована в виде иерархии, которая начинается с корневой директории "/". Все файлы и директории располагаются внутри неё или в её поддиректориях.
В корневой директории находятся системные файлы и директории, такие как:
"/bin"
"/etc"
"/dev"
"/home"
"/usr"
Каждая из этих директорий имеет своё назначение и содержит определённые файлы. Иерархия файловой системы в Linux позволяет организовать файлы и директории в логически связанные группы, что упрощает работу с ними и обеспечивает стабильную работу системы.


========== Файлы, индексные дескрипторы, блоки данных =========================================
Файлы, индексные дескрипторы и блоки данных — это основные составляющие файловой системы Linux.
Файлы 
- в Linux являются основными единицами хранения информации. Они могут содержать текстовые данные, изображения, звуковые файлы, программные коды и другие типы информации. Каждый файл имеет уникальное имя и специальный индексный дескриптор, который используется для доступа к файлу.
Попробуйте создать файл в своём домашнем каталоге командой:
touch test1.txt

Индексный дескриптор (inode) 
- представляет собой числовое значение, которое идентифицирует файл. Этот индексный дескриптор используется ядром Linux для доступа к файлу. Когда вы запрашиваете информацию из файла, ядро Linux использует индексный дескриптор, чтобы найти нужный файл на диске.
Выполнив команду:
ls -i test1.txt 		вы увидите, какой inode присвоился вашему файлу
ls -li {directory}		

Блок данных 
— это фактические данные, которые хранятся в файле. Каждый файл делится на блоки данных по размеру в несколько Кб. Блоки данных хранятся на диске и являются основными составляющими файловой системы Linux. Когда вы запрашиваете информацию из файла, ядро Linux читает блоки данных из диска в оперативную память и передаёт их вам.
Выполните следующую команду:
echo "Hello world!" > test1.txt
echo выводит в терминал введённую информацию
> - этот знак в linux используется для перенаправления вывода команды в файл или другую команду
Для просмотра блока данных файла в Linux можно использовать команду hexdump.  
hexdump -C 1.txt		Эта команда выводит содержимое файла в шестнадцатеричном формате, что позволяет увидеть блоки данных.

============ Типы файлов ls -l ==================
Основные типы файлов включают:
1	Обычные файлы, - содержат данные в текстовом или исполняемом формате;
2	Каталоги, - специальные файлы, которые содержат список других файлов и каталогов;
3	Символические ссылки, - файлы, которые указывают на другие файлы или каталоги;
4	Устройства - файлы, которые представляют собой физические или виртуальные устройства, такие как жёсткие диски, принтеры и т. д.
Файлы бывают следующих типов. Тип файла можно увидеть в первом символе вывода
ls -l:
( - ) обычный файл
( d ) директория — это объект файловой системы, который упрощает работу с файлами, позволяя группировать их
( l ) символическая cсылка
( b ) файл блочного устройства
( c ) файл символьного устройства
( p ) именованные каналы
( s ) сокет
( n ) сетевой файл


============= Стандарт иерархии фс (FHS - Standart) ====================
Стандарт иерархии файловой системы (FHS) определяет структуру и расположение файлов в Linux. Он описывает назначение каждого каталога в системе и устанавливает правила для размещения файлов в каждом каталоге. FHS обеспечивает совместимость между различными дистрибутивами Linux и упрощает портирование программ между ними.

Назначение основных каталогов
"/bin" — содержит основные исполняемые файлы системы, которые используются для запуска основных команд
"/etc" — содержит конфигурационные файлы системы
"/dev" — содержит файлы устройств, которые представляют собой интерфейс к оборудованию компьютера
"/home" — содержит домашние директории пользователей
"/usr" — содержит системные файлы и библиотеки, которые используются при работе программ

Кроме того, в Linux существуют специальные директории, которые имеют особое назначение:
"/proc" — хранит информацию о текущих процессах и состоянии системы, - виртуальная файловая система - отображение процессора в файле.
"/sys" — хранит информацию о конфигурации системы и устройствах
"/tmp" — хранит временные файлы, которые создаются при работе программ
"/var" — хранит переменные данные, такие как логи, кэши и т. д.

Каждый файл и директория в Linux имеет свой уникальный путь от корневой директории. 
Абсолютный путь 	полный путь до файла / то есть например, путь к файлу "passwd" будет выглядеть так: "/etc/passwd".
Относительный путь	./file


============== Команды навигации по файловой системе ========================
cd 	— используется для перехода в другой каталог.
cd ~ 	— переход в домашнюю директорию пользователя под которым выполняется команда cd .. — переход в предыдущую директорию относительно текущей 
pwd 	— показывает текущий каталог
ls 	— показывает список файлов в текущем каталоге
mkdir 	— создаёт новый каталог. Чтобы создать вложенный каталог внутри создаваемого, используйте ключ «-p»,  а чтобы создать несколько — фигурные скобки с перечислением через запятую{..,..}. Например, создадим каталоги test1 test2 и test3 внутри каталога 
mkdir -p /home/admuser/data/{test1,test2,test3}		серия директорий
mkdir -p /home/admuser/data/{test1,test2,test3}/into	серия директорий, со вложенной директорией в каждой
mkdir -p /home/admuser/data/test1/test2/test3		вложенные директории
rmdir -R	удаление директории

============== Команды создания файлов ======================================
touch 	— создаёт новый пустой файл
echo 	— используется для записи текста в файл
cat 	— используется для объединения нескольких файлов в один

============== Операции с файлами ============================================
cp 			— копирует файлы из одного каталога в другой
cp -r {from} {to}	- со всеми вложенными, при копировании директории
mv 			— перемещает файлы из одного каталога в другой. Может без -r
rm 			— удаляет файлы
rm -r 			- со всеми вложенными

============== Поиск файлов ==================================================
find 	— используется для поиска файлов по различным критериям, таким как имя файла, размер, дата изменения и т. д.
Например, найдём наш файл test1.txt командой 
find /home -type f -name "test1.txt"
find ~ -type d 		( d ) директория
find ~ -type s		( s ) сокет
grep — используется для поиска текста в файлах

============== Использование менеджера файлов и Midnight Commander ===========
Менеджер файлов 
– это графический интерфейс для управления файлами и каталогами. Он позволяет быстро просматривать содержимое каталогов, копировать, перемещать и удалять файлы. В Astra Linux менеджером файлов является программа fly-fm. fly-fm предназначена для просмотра папок рабочего стола и элементов файловой системы и выполнения основных функций управления файлами. Позволяет подключать и отключать файловой системе носителей доступных устройств хранения данных, таких как локальные жёсткие диски и их разделы, компакт- и DVD-диски, USB-накопители. Также позволяет обращаться к сетевым Samba-ресурсам.

Midnight Commander 
– это текстовый менеджер файлов, который позволяет управлять файлами и каталогами из командной строки. Он имеет мощные возможности поиска и фильтрации файлов, а также поддерживает работу с архивами.
