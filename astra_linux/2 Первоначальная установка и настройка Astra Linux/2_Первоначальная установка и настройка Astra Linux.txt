========== Уровни защищённости ОС Astra Linux ===============
При установке ОС далее вам нужно выбрать уровень защищённости в зависимости от приобретённой лицензии.

В Astra Linux выделяют 3 уровня защищённости ОС: 
1) базовый уровень — «Орёл» 
2) усиленный уровень — «Воронеж» 
3) максимальный уровень — «Смоленск»

«Орёл» — предоставляет базовый уровень защищённости и предназначен для всех сотрудников, работающих на устройствах, которым не требуется аттестация по безопасности.
«Воронеж» и «Смоленск» — сертифицированные варианты ОС.
«Воронеж» — подходит для работы в любых государственных информационных системах и на объектах КИИ.
«Смоленск» — для работы с документами особой важности, в том числе с грифом «Совершенно секретно».

Функции безопасности уровня «Орёл»:
Запрет вывода меню загрузчика будет запрещён вывод меню загрузчика GRUB 2 в процессе загрузки будет загружаться ядро ОС, выбранное по умолчанию
Запрет трассировки ptrace будет отключена возможность трассировки и отладки выполнения программного кода
Запрос пароля для команды sudo будет включено требование ввода пароля при использовании механизма sudo
Запрет установки бита исполнения будет включён режим запрета установки бита исполнения, обеспечивающий предотвращение несанкционированного создания пользователями исполняемых сценариев для командной оболочки
Запрет исполнения скриптов пользователя будет блокировано интерактивное использование пользователем интерпретаторов
Запрет исполнения макросов пользователя будет блокировано исполнение макросов в стандартных приложениях
Запрет консоли будет блокирован консольный вход в систему для пользователя и запуск консоли из графического интерфейса сессии пользователя
Системные ограничения ulimits будут включены системные ограничения, установленные в файле /etc/security/limits.conf
Запрет автонастройки сети будет отключена автоматическая настройка сети в процессе установки ОС, сеть необходимо будет настроить вручную в соответствии с рекомендациями настоящего руководства
Местное время для системных часов будет включён режим интерпретации показаний аппаратных (RTC) часов

Функции безопасности уровня «Воронеж» (доступны все функции уровня «Орёл»):
Мандатный контроль целостности будет включён механизм мандатного контроля целостности
Замкнутая программная среда будет включён механизм, обеспечивающий проверку неизменности и подлинности загружаемых исполняемых файлов формата ELF
Очистка освобождаемой внешней памяти будет включён режим очистки блоков ФС непосредственно при их освобождении, а также режим очистки разделов страничного обмена 

Функции безопасности уровня «Смоленск» (доступны все функции уровней «Орёл» и «Воронеж»):
Мандатное управление доступом будет включен механизм мандатного управления доступом


========== Процесс загрузки ОС ====================
Процесс загрузки Linux-системы и Astra Linux, в частности, можно разделить на несколько этапов:
1. BIOS (Basic Input/Output System) 
Это программа, которая запускается при включении компьютера и проверяет аппаратное обеспечение на наличие возможных проблем. Затем BIOS ищет загрузочное устройство (обычно жесткий диск или USB-флешку).

2. MBR (Master Boot Record) 
Это первый сектор жёсткого диска, где хранится загрузочный заголовок, который указывает на местоположение GRUB (Grand Unified Bootloader) — загрузчика ОС. GRUB представляет собой программу, которая загружает ядро операционной системы.

3. Kernel (ядро Linux) 
Ядро Linux загружается в оперативную память компьютера. Оно отвечает за основную работу системы, используя написанные программы и драйверы для управления оборудованием, работой многих процессов и многое другое.

4. Init 
Это процесс, который запускается ядром и имеет идентификатор процесса PID 1. Он выполняет инициализацию системы, запускает сервисы и демоны, а также запускает процесс загрузки пользователя.

5. Runlevel Linux 
использует runlevel для определения состояния системы. Runlevel определяет доступные сервисы и демоны для запуска в операционной системе в текущее время.

6. User Space 
После того, как система загрузится, пользовательский интерфейс будет запущен, и пользователь сможет начать использовать систему на своё усмотрение.
