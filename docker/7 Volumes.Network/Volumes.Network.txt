VOLUMES					- абстракция, директория на Host_Machine, где сохраняются данные из контейнера.
Средство персистентного хранения между запусками контейнера.
persistent storage 			— это система хранения данных, которая сохраняет информацию даже после отключения питания. Также может называться энергонезависимым хранением. 
Особенность персистентного хранения 	— сохранение предыдущих состояний структуры данных при внесении изменений. Это позволяет: 
	обращаться к любой версии структуры;
	изменять не только последнюю, но и любую версию.
Виды персистентного хранения 		- Существует несколько уровней персистентности: 
	Частичная	— к каждой версии можно делать запросы, но изменять можно только последнюю.
	Полная 		— можно менять не только последнюю, но и любую версию, также к любой версии можно делать запросы.
	Конфлюэнтная 	— позволяет объединять две структуры данных в одну (например, деревья поиска, которые можно сливать).
	Функциональная 	— полностью персистентна по определению, так как в ней запрещаются уничтожающие присваивания: любой переменной значение может быть присвоено только один раз, изменять значения переменных нельзя.
Технологии персистентного хранения	- для реализации персистентного хранения используются, например:
Системы управления базами данных (СУБД) 			— используют комбинацию методов «грязной записи» и ведения журнала транзакций, которые обеспечивают сохраняемость.
Распределённые файловые системы или блочные сетевые хранилища 	— прикреплены ко всем узлам, на которых работают контейнеры, и обеспечивают доступ к томам для персистентного хранения данных.
Локальные персистентные тома (PV) 				— предоставляют возможность хранения сырых данных, а приложения, работающие поверх этих PV, самостоятельно собирают информацию, позволяющую масштабировать данные.
Применение персистентного хранения	-  персистентное хранение используется в различных приложениях, например:
	Базы данных, 
	объектные хранилища, 
	хранилища ключей и значений 	— данные хранятся независимо от объектов и программ, которые работают с ними, что делает их доступными для других приложений как общий ресурс. 
Приложения для редактирования документов — персистентное хранение исторических версий делает частые операции (например, «отменить» и «повторно») более эффективными. 
Облачно-ориентированные приложения 	— персистентное хранение данных в качестве резервного хранилища, что удобно для масштабирования данных и управления растущими требованиями к данным.
